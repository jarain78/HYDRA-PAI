1 Configuration and Environment

The script starts by importing the required libraries and setting critical constants.

  • YOLO_TO_ONTO: It is essential because it translates the label seen by the AI (e.g., "cup") into the exact individual name in the OWL ontology.
  • RoomZone: An immutable dataclass that defines invisible boxes (X, Y limits) in 3D space for the "kitchen" and "bathroom", along with their exact drop points (drop_xy).
  • Functions: room_from_xy checks mathematically which zone an object falls into.

# Definition of Rooms / Physical zones of the simulator
# ============================================================
@dataclass(frozen=True)
class RoomZone:
    name: str
    xmin: float; xmax: float
    ymin: float; ymax: float
    drop_xy: Tuple[float, float]

ROOMS = [
    # Defines the kitchen zone and where to drop objects (0.10, 0.18)
    RoomZone(name="kitchen",  xmin=-0.15, xmax=0.25, ymin=-0.25, ymax=0.25, drop_xy=(0.10,  0.18)),
    RoomZone(name="bathroom", xmin=0.25,  xmax=0.65, ymin=-0.25, ymax=0.25, drop_xy=(0.52, -0.18)),
]

def room_from_xy(x: float, y: float) -> Optional[str]:
    # Iterates over zones; if XY is within bounds, returns the name.
    for r in ROOMS:
        if r.xmin <= x <= r.xmax and r.ymin <= y <= r.ymax:
            return r.name
    return None

2 Ontology Engine

Here the system reads the .owl (Web Ontology Language) file using OWLReady2.

  • load_ontology_index: A critical function. It loads the file and extracts all instances of the "Object" class.
  • Property Extraction: For each object, it reads semantic relations:
    1. locatedIn -> Extracts its destination.
    2. canBePickedBy -> If it has an assigned robot, it is assumed pickable (pickable = True).
ontology_parser.py 
@dataclass
class OntoIndexEntry:
    instance: object
    room: Optional[str]       # Destination according to 'locatedIn'
    robot: Optional[str]      # Assigned robot in 'canBePickedBy'
    pickable: bool            # Boolean derived from whether it has a robot or not

def load_ontology_index(onto_path: str):
    # Loads the OWL file into memory
    onto = get_ontology(onto_path).load()
    ObjectClass = onto.search_one(iri=IRI_OBJECT)

    info = {} # Dictionary for O(1) access
    # Iterates over all instances of the Object class in the ontology
    for obj in ObjectClass.instances():
        name = obj.name.lower()
        
        # If the object has the 'locatedIn' property, store the first room
        room = None
        if hasattr(obj, "locatedIn") and obj.locatedIn:
            room = obj.locatedIn[0].name.lower()

        # If the object can be picked by someone, store who
        robot = None
        if hasattr(obj, "canBePickedBy") and obj.canBePickedBy:
            robot = obj.canBePickedBy[0].name

        # If the robot exists, the object is physically manipulable
        pickable = bool(robot)
        
        # Store the structured record
        info[name] = OntoIndexEntry(instance=obj, room=room, robot=robot, pickable=pickable)

    return onto, info

3 CLIPS Wrapper

The bridge between Python and the expert engine written in C.

  • build_clips_planner: Takes the string containing the LISP rules (defrule out-of-place-move, etc.), writes it to a temporary file, and loads it into clipspy via env.batch_star().
  • clips_get_tasks: Iterates over CLIPS working memory (facts), finds those whose template is "task" (the resulting command), and extracts them into Python dictionaries that the robotic arm can interpret.

def clear_clips_dynamic(env: clips.Environment) -> None:
    # Clears facts each frame to avoid accumulating garbage from previous frames
    to_retract = []
    for f in env.facts():
        if f.template.name in {"obj", "task", "trace"}:
            to_retract.append(f)
    for f in to_retract:
        f.retract()

def clips_get_tasks(env: clips.Environment) -> List[Dict]:
    # Extracts outputs (tasks) generated by the inference engine into Python
    out = []
    for f in env.facts():
        if f.template.name == "task":
            out.append({
                "action": str(f["action"]),
                "id": str(f["id"]),
                "to": str(f["to"]),
                "reason": str(f["reason"]),
            })
    return out

4 Robot Control (Panda)

Translates CLIPS conceptual tasks into 3D motor motions.

  • panda_move_ee: Computes Inverse Kinematics (IK). Given a 3D XYZ point, it returns the exact joint angles for the 7 arm joints to reach it, interpolating smoothly.
  • grasp_with_constraint: In PyBullet, physically grasping an object can be unstable. This function creates a "Constraint" (an invisible joint) between the End Effector (gripper) and the object.
robot_kinematics.py

5 The Central Brain (Main)

The infinite loop that ties the entire AI pipeline together.

  • 1. Perception: The camera takes a snapshot, YOLO processes it. Pixels (u, v) are projected to PyBullet 3D coordinates.
  • 2. Ontology Injection: For each detected box, the ontology is queried to determine pickable and dst_room.
  • 3. Inference: Strings are assembled and sent to clips_env.assert_string(), then .run().
  • 4. Execution: If CLIPS returns a move task, the kinematics script is called to execute execute_pick_and_place.
